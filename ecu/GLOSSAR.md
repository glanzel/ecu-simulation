# Kurzglossar Benennung (Domäne)


| Code / Variable / Funktionsname | Bedeutung                                                                                                                                                                                  | Bezeichnung UI               | Auch / anders benannt                                                                                                     |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `vej_ziel`                      | Langfristiges planetares Ziel je Grenze (Jahres-Obergrenze, physische Verschmutzungseinheiten/a).                                                                                          | VEJ-Ziel                     | z. B. `vej_ziel["co2"]` (`vej_jahr_soll_lang`)                                                                            |
| `vej_soll`                      | Kurzfristiges Soll (Jahr); im aktuellen Modell **identisch** zu `vej_ziel` — kein separates Datenfeld.                                                                                     | VEJ-Soll (Jahr)              | im Code kein eigenes Feld; (`vej_jahr_soll_kurz` — Vorschlag)                                                             |
| `vet_soll`                      | Kurzfristiges Soll pro Monat = `vej_ziel / 12` (Monats-Obergrenze je Grenze).                                                                                                              | VET-Soll                     | z. B. `vet_soll["co2"]`                                                                                                   |
| `vej_ist`                       | Beobachteter bzw. modellierter Verbrauch pro Monat (gleiche Einheit wie `vet_soll`).                                                                                                       | VEJ-Ist                      | z. B. `vej_ist["co2"]`                                                                                                    |
| `fraction_of_vej_ziel`          | Anteil des VEJ-Ziels (Referenznachfrage / Startanker), dimensionslos 0…1.                                                                                                                  | Start-Nachfrage (Anteil VEJ) | —                                                                                                                         |
| `demand_at_reference_price`     | Nachfragemenge bei Referenz-Schattenpreis (pro Monat je Grenze); Eingang der isoelastischen Kurve (`consumption_quantity`), inkl. Wachstum/Rauschen aus der Simulation.                    | demand                       | z. B. `demand_at_reference_price["co2"]`; in `ConsumptionRecord` optional                                                 |
| `consumption_quantity`          | Aus `demand_at_reference_price` und Preisverhältnis berechnete Menge zum aktuellen `shadow_price` (Vorstufe); danach ggf. `apply_consumption_budget` → gespeicherter Ist-Konsum `vej_ist`. | consum (Kurve)               | Funktion in `simulation/demand.py`; Rohwerte `_raw_vej_ist_at_prices`                                                     |
| `ecumenge_ziel_J`               | Konfiguriertes langfristiges Jahresziel (ECU/Jahr); Referenz für Slack ggü. Kontenrahmen `bundle_ecu` und für Preisnormierung Σ p·VEJ-Ziel.                                                   | ECU-Ziel (Jahr)              | `SimulationConfig.ecumenge_ziel_J`; Timeline-Feld gleicher Name; `ecumenge_ziel_J_konfig` = unveränderte Konfigkopie       |
| `bundle_ecu`                    | Kontenrahmen: Σ p_i·VEJ-Ziel_i zu den gültigen Schattenpreisen (jährlicher ECU-Wert des „vollen Ziel-Bündels“; Zeitreihe im Report — Preis des jeweiligen Monats). Im ECU-Diagramm zusätzlich als **Monatsanteil** (÷12).               | Ecu VEJ-Soll ges. (Jahr)     | `PeriodResult.bundle_ecu`; weicher Pfad: schrittweise Annäherung an `ecumenge_ziel_sim_J`, nicht jeden Monat exakt `ecumenge_ziel_J`. |
| `ecumenge_ziel_sim_J`           | Simuliertes langfristiges Jahresziel (weicher Pfad: wirksamer Wert nach Ratchet; sinkt höchstens um p %/Periode bis `ecumenge_ziel_J_konfig`).                                             | ECU-Ziel sim. (Jahr)         | `ratchet_ecumenge_ziel_sim_J`; monatlich ÷12 als Referenz (`warmup_diag_ecumenge_ziel_sim_monthly`, Diagramm `ecumenge_ziel_sim_J_T`) |
| `ecumenge_J`                    | Simulierte wirksame Jahresmenge (Start bzw. Ausgang für die Budgetlogik; bei hoher Start-Auslastung ≥ `ecumenge_ziel_J`).                                                                  | ECU-Menge J (sim.)           | `ecumenge_J_from_start`; erste Periode: Grundlage für `ecumenge_T`/`12`                                                    |
| `ecumenge_T`                    | Im Zeitschritt simuliert ausgegebene ECU-Menge (Budgetobergrenze für Σ p·c in dieser Periode).                                                                                               | ECU-Menge T (sim.)           | `ecumenge_T_override` (weicher Pfad); frühere Sprechweise „ECU-Decke“ / Monats-Cap                                         |
| `ecu_ist_T`                     | Tatsächlich verbuchte ECU im Zeitschritt (Σ p·VEJ-Ist zum gespeicherten Konsum).                                                                                                             | ECU-Ist (Monat)              | gegenüber `ecumenge_T`; Grenztabelle: Spalte **Ecu VEJ-Ist** = p·VEJ-Ist je Grenze                                       |


**UI / Tabellen:** Lesbare Kurzbezeichnungen ohne Unterstriche (siehe Spalte *Bezeichnung UI*); in der CLI v. a. in Tabellenlegenden (`cli_simulation.py`). Wo die Oberfläche die Code-Symbole zeigt, entsprechen sie den Namen in der ersten Spalte (`ecumenge_*`, `ecu_ist_*`, Suffix `_J` = Jahr, `_T` = Zeitschritt).
